# Issue Tracking

The repository currently uses local Markdown as its authoritative work tracker. No Git remote or external issue tracker is configured.

The governing work item is `docs/specs/canonical-research-flywheel.md`. Its progress ledger records phase state, acceptance evidence, blockers, and next actions. Exact implementation steps live in `docs/plans/canonical-research-flywheel-implementation.md`.

Do not publish or synchronize external issues unless the user explicitly selects a tracker. If a tracker is selected later, mirror the local spec rather than replacing it.
