# Domain Documentation

This repository uses a single-context documentation layout.

- `CONTEXT.md` defines canonical domain terms and invariants.
- `docs/adr/` records load-bearing architecture decisions.
- `docs/specs/canonical-research-flywheel.md` defines desired outcomes and acceptance criteria.
- `docs/plans/canonical-research-flywheel-implementation.md` defines current execution slices.

Agents must read these documents before changing the RunEngine, evaluation law, promotion law, evidence schema, lifecycle, or campaign control plane. Update the governing spec when verified work changes progress or discovers an unknown unknown.
